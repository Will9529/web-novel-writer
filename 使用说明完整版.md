## ✨ 核心功能

| 标签页 | 功能 | 适用场景 |
|--------|------|----------|
| 📜 风格宪法 | 定义写作风格，样章自动提炼 | 项目启动时建立风格规范 |
| 📝 章纲编写 | 六要素结构化章纲，AI 辅助 | 每章写作前规划 |

| ✍️ 写作助手 | 智能编辑器，AI 辅助写作 | 日常写作主力工具 |
| 📖 章节生成 | 五层 Prompt 一键生成 | 快速生成初稿 |
| 🔍 质检中心 | 三道检查，质量把关 | 生成后或写作后质检 |
| 📊 进度看板 | 项目数据统计 | 随时查看写作进度 |

## 🚀 快速开始（3 分钟）

### 步骤 1：安装 Python
- 下载：https://www.python.org/downloads/
- ⚠️ 安装时勾选 **"Add Python to PATH"**

### 步骤 2：获取 API Key
- 访问：https://dashscope.console.aliyun.com/
- 登录阿里云 → API-KEY 管理 → 创建新 Key

### 步骤 3：配置项目
```bash
# 1. 打开项目文件夹
cd web_novel_writer

# 2. 编辑 config.yaml
# 将 api_key: "" 改为 api_key: "sk-你的 Key"

# 3. 安装依赖
pip install -r requirements.txt

# 4. 启动应用
python main.py
```

或双击 **`install.bat`** 自动安装，然后双击 **`start.bat`** 启动

## 📖 详细使用流程

### 第一阶段：项目初始化（15 分钟）

```
1. 启动应用 → 新建项目
   ↓
2. 📜 风格宪法标签页
   - 上传 2-3 章满意样章 → 点击"从样章自动提炼风格"
   - 或手动编写风格规范
   - 点击"保存风格宪法"
   ↓
3. 📝 章纲编写标签页
   - 输入章节号（如第 1 章）
   - 填写六要素或点击"AI 辅助填充"
   - 点击"保存章纲"
   ↓
4. 🌍 世界状态标签页
   - 初始化世界状态（主角、配角、反派设定）
   - 点击"保存"
```

### 第二阶段：日常写作（每章 30-60 分钟）

```
方式 A：AI 生成初稿 + 人工修改
─────────────────────────────────────
1. 📖 章节生成标签页
   - 选择章节号 → 点击"开始生成"
   - 等待 1-2 分钟 → 查看生成结果
   - 点击"保存章节"
   ↓
2. ✍️ 写作助手标签页
   - 加载已生成章节
   - 手动修改润色
   - 使用 AI 辅助功能（续写/润色/扩写）
   ↓
3. 🔍 质检中心
   - 选择章节 → 点击"开始质检"
   - 查看得分和问题清单
   - 根据建议修改
   ↓
4. 🌍 世界状态
   - 更新本章状态变更
   - 点击"生成状态变更 Diff"
   - 保存

方式 B：人工写作 + AI 辅助
─────────────────────────────────────
1. ✍️ 写作助手标签页
   - 选择章节号 → 开始写作
   - 卡顿时使用 AI 辅助：
     * 智能续写：根据上文续写 500 字
     * 润色段落：优化选中段落文笔
     * 扩写场景：增加感官细节
     * 生成对话：根据场景生成对话
   ↓
2. 自动保存（60 秒一次）或手动保存
   ↓
3. 🔍 质检中心 → 质检本章
   ↓
4. 🌍 世界状态 → 更新状态
```

### 第三阶段：进度追踪（随时）

```
📊 进度看板标签页
- 查看已完成章节数、总字数
- 检查章纲完成率
- 查看情节债务（伏笔）统计
- 点击"刷新"更新数据
```

## 🛠️ 功能详解

### ✍️ 写作助手 - AI 辅助功能

| 功能 | 使用方法 | 效果 |
|------|----------|------|
| 智能续写 | 输入部分内容 → 点击"智能续写" | 续写 500 字，保持风格一致 |
| 润色段落 | 选中段落 → 点击"润色段落" | 优化句式，增强画面感 |
| 扩写场景 | 选中场景描述 → 点击"扩写场景" | 增加到 300 字，添加感官细节 |
| 生成对话 | 选中场景说明 → 点击"生成对话" | 生成 3-5 轮自然对话 |

### 📖 章节生成 - 五层 Prompt

```
第 1 层：风格宪法
  └─ 叙事视角、句式偏好、描写密度、禁用词表...

第 2 层：世界状态摘要
  └─ 人物状态、情节债务、已用模板...

第 3 层：上章末尾 200 字
  └─ 确保情节连贯

第 4 层：当章章纲
  └─ 场景、人物、核心事件、开场/结尾钩子...

第 5 层：生成指令
  └─ 字数要求、格式要求、特殊限制...
```

### 🔍 质检中心 - 评分标准

| 得分 | 等级 | 建议 |
|------|------|------|
| 90-100 | ✅ 优秀 | 可直接使用 |
| 70-89 | ⚠️ 良好 | 小修后可用 |
| <70 | ❌ 需修改 | 根据建议修改后复检 |

**扣分项：**
- 禁用词命中：-20 分
- 情节债务未填：-20 分
- 结构相似度>60%：-15 分
- 字数<2500：-10 分
- 格式问题：-5 分

## ⚙️ 配置说明

### config.yaml 完整配置

```yaml
# ========== 阿里云 API 配置 ==========
aliyun:
  api_key: "sk-xxxxxxxxxxxx"        # 【必填】API Key
  secret_key: ""                     # 【可选】一般不需要
  model: "qwen-plus"                 # 模型选择：
                                     # - qwen-turbo: 快速便宜
                                     # - qwen-plus: 平衡（推荐）
                                     # - qwen-max: 效果最好

# ========== 项目设置 ==========
project:
  default_chapter_words: 2750       # 目标每章字数
  quality_check_enabled: true       # 启用质检
  auto_save: true                   # 自动保存

# ========== 禁用词表 ==========
banned_words:
  - 突然      # 可添加更多
  - 瞬间
  - 顿时
  - 不禁
  - 只见
```

## ❓ 常见问题

### Q: 双击 bat 文件闪退？
**A:** 右键 → 以管理员身份运行，或打开 cmd 拖入文件执行

### Q: 依赖安装失败？
**A:** 检查网络连接，或手动执行：
```bash
pip install PyQt6 qianfan PyYAML markdown requests
```

### Q: 提示 API Key 错误？
**A:** 
1. 确认 config.yaml 中 api_key 已填写
2. 确认 Key 格式正确（sk-开头）
3. 确认阿里云账号有余额

### Q: 生成内容不理想？
**A:**
1. 检查风格宪法是否详细
2. 检查章纲六要素是否完整
3. 尝试更换模型（qwen-max 效果更好）

### Q: 如何打包成 exe 发给别人？
**A:**
1. 双击 `build.bat`
2. 等待打包完成
3. 从 `dist/` 文件夹获取 exe
4. 将 config.yaml 复制到 exe 同目录
